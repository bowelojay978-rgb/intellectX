# Security policy

## Reporting a vulnerability

Do not report vulnerabilities through public GitHub issues, pull requests, screenshots, or logs. Use a private maintainer channel when one is available. If no private channel is available, avoid sharing exploit details publicly and ask a repository maintainer how to report the issue safely.

> [!WARNING]
> Do not create issues for potential security vulnerabilities. Issues are public and can be seen by potentially malicious actors.

Thanks for helping to keep IntellectX safe.
